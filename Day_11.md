# 📘 Day 11: Functions (Revision Notes)

## What are Functions?

* Reusable blocks of code that perform a specific task.
* Defined using the `def` keyword.
* Execute only when called.

---

## Function Types

* Function without parameters
* Function with parameters
* Function with default parameters
* Function with arbitrary positional arguments (`*args`)
* Function with arbitrary keyword arguments (`**kwargs`)

---

## Parameters vs Arguments

* **Parameter:** Variable defined in a function.
* **Argument:** Actual value passed when calling the function.

---

## Return Statement

* Returns a value from a function.
* If omitted, the function returns `None`.
* Can return any data type (string, number, boolean, list, etc.).

---

## Keyword Arguments

* Pass arguments using parameter names.
* Order of arguments does not matter.

---

## Default Parameters

* Provide default values for parameters.
* Used when no argument is supplied.

---

## Arbitrary Positional Arguments (`*args`)

* Accepts any number of positional arguments.
* Stored as a tuple.

---

## Arbitrary Keyword Arguments (`**kwargs`)

* Accepts any number of keyword arguments.
* Stored as a dictionary.

---

## Dictionary Unpacking

* Use `**dictionary` to pass dictionary values as keyword arguments.
* Dictionary keys must match the parameter names.

---

## Functions as Arguments

* Functions can be passed to other functions.
* Enables reusable and flexible code.

---

# Key Points to Remember

* `def` → Define a function.
* `return` → Send a value back to the caller.
* `print()` displays output; `return` provides output for further use.
* Parameters are placeholders; arguments are actual values.
* Keyword arguments can be passed in any order.
* Default parameters make arguments optional.
* `*args` stores extra positional arguments as a tuple.
* `**kwargs` stores extra keyword arguments as a dictionary.
* `**dictionary` unpacks a dictionary into keyword arguments.
