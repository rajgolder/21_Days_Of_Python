# 📘 Day 9: Conditionals (Revision Notes)

## Conditionals

* Used to make decisions based on conditions.
* Control the flow of program execution.
* Main conditional statements:

  * `if`
  * `if...else`
  * `if...elif...else`

---

## `if` Statement

* Executes a block only if the condition is `True`.

---

## `if...else`

* Executes one block if the condition is `True`.
* Executes another block if the condition is `False`.

---

## `if...elif...else`

* Used to check multiple conditions.
* Conditions are evaluated from top to bottom.
* Only the first matching condition executes.

---

## Ternary (Short Hand) Condition

* One-line conditional expression.
* Used for simple decisions.

---

## Nested Conditions

* An `if` statement inside another `if`.
* Useful for checking dependent conditions.
* Can often be simplified using logical operators.

---

## Logical Operators

### `and`

* Returns `True` only if **all** conditions are `True`.

### `or`

* Returns `True` if **at least one** condition is `True`.

### `not`

* Reverses the result of a condition.
* `True` becomes `False` and vice versa.

---

## Comparison Operators

* `==` → Equal to
* `!=` → Not equal to
* `>` → Greater than
* `<` → Less than
* `>=` → Greater than or equal to
* `<=` → Less than or equal to

---

## Truthy & Falsy Values

### Falsy Values

* `False`
* `None`
* `0`
* `0.0`
* `''` (empty string)
* `[]` (empty list)
* `()` (empty tuple)
* `{}` (empty dictionary)
* `set()`

### Truthy Values

* Any non-empty string, list, tuple, dictionary, or set.
* Any non-zero number.
* `True`

---

# Key Points to Remember

* `if` checks a condition.
* `else` runs when all previous conditions are `False`.
* `elif` checks additional conditions.
* Conditions are evaluated in order.
* Use `and` when all conditions must be true.
* Use `or` when any one condition is enough.
* Use `not` to invert a condition.
* Nested conditions can often be replaced with logical operators.
* Indentation defines the scope of each conditional block.
